wiki-tokenise

tokenisation of real-world assets · start here

An Obsidian vault collating authoritative sources on the tokenisation of real-world assets, with five browser artefacts that read the vault directly. Sister vault: wiki-ai-thinkers.

Launch the browser artefacts

The artefacts live under .workspace/ and need a local HTTP server (Chrome refuses the File System Access flow over file://). One Python launcher, three OS wrappers.

OSHow to launch
WindowsOpen .workspace/ in File Explorer (Win+E) and double-click serve.bat.
macOS / Linuxcd .workspace && ./serve.sh
Any platformpython .workspace/serve.py · default port 8766

A console window opens, prints a URL, and waits. Ctrl-click the URL or paste it into Chrome / Edge. The landing page (index.html) is the front door.

The five artefacts

01graph · force-directed graph of every wikilink.
02convergence · jurisdiction × doctrine matrix.
03timeline · events on a horizontal time axis, in lanes.
04realised vs projected · forecasts vs realised AUM.
05compare · pick two notes, see frontmatter and body side by side.

Requirements

  • Python 3 on the system PATH (Windows: python; macOS/Linux: python3).
  • Chrome 86+ or Edge 86+ for the File System Access API. Firefox and Safari are not supported.
  • iA Writer fonts (auto-installed to your user font registry on first launch via PowerShell).

Weekly refresh

The /wiki-tokenise weekly-refresh command runs the watchlist monitor across 12 source channels, syncs new sources into NotebookLM, then refreshes MOCs. Pilot channel: BIS Working Papers. Scheduling guidance is in 99_Meta/VAULT-DESIGN.md §5.4.
